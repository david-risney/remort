# Data Model: RDP Connect & Disconnect

**Feature**: 001-rdp-connect-disconnect
**Date**: 2026-03-22

## Entities

### ConnectionState (Enum)

Represents the lifecycle state of an RDP connection. Drives UI element availability and status display.

| Value | Description | UI Behavior |
|-------|-------------|-------------|
| `Disconnected` | No active session. Initial state and state after clean/error disconnect. | Connect enabled, Disconnect disabled, hostname editable |
| `Connecting` | Connection attempt in progress (after `Connect()`, before `OnConnected`). | Connect disabled, Disconnect enabled (cancels attempt), hostname read-only |
| `Connected` | Active RDP session rendering in the window. | Connect disabled, Disconnect enabled, hostname read-only |

**Transition triggers**:

| From | To | Trigger |
|------|----|---------|
| Disconnected | Connecting | User clicks Connect (valid hostname) |
| Connecting | Connected | `IRdpClient.Connected` event fires |
| Connecting | Disconnected | `IRdpClient.Disconnected` event fires (auth cancel, unreachable host, error) |
| Connected | Disconnected | User clicks Disconnect OR `IRdpClient.Disconnected` event fires (remote drop, network loss) |

### IRdpClient (Interface вҖ” Connection/IRdpClient.cs)

Abstracts the COM-based `RdpClientHost` for ViewModel consumption. This is the boundary between the Connection domain and the Interop layer.

```csharp
public interface IRdpClient
{
    // Properties вҖ” set before Connect()
    string Server { get; set; }
    int DesktopWidth { get; set; }
    int DesktopHeight { get; set; }

    // State
    bool IsConnected { get; }

    // Operations
    void Connect();
    void Disconnect();

    // Configuration
    void ApplyDefaultSettings();

    // Disconnect reason
    int ExtendedDisconnectReason { get; }
    string GetErrorDescription(int disconnectReason, int extendedReason);

    // Events (CLR events raised from COM callbacks)
    event EventHandler? Connecting;
    event EventHandler? Connected;
    event EventHandler<DisconnectedEventArgs>? Disconnected;
}
```

### MainWindowViewModel (ViewModel вҖ” Connection/MainWindowViewModel.cs)

| Property | Type | Source Generator | Description |
|----------|------|-----------------|-------------|
| `Hostname` | `string` | `[ObservableProperty]` | User-entered hostname. Bound to TextBox. Trimmed before use. |
| `ConnectionState` | `ConnectionState` | `[ObservableProperty]` | Current state enum. Drives CanExecute and status. |
| `StatusText` | `string` | `[ObservableProperty]` | Human-readable status ("Disconnected", "Connecting to XвҖҰ", "Connected to X", "Disconnected: reason"). |

| Command | Source Generator | CanExecute Condition |
|---------|-----------------|---------------------|
| `ConnectCommand` | `[RelayCommand(CanExecute = nameof(CanConnect))]` | `ConnectionState == Disconnected && !string.IsNullOrWhiteSpace(Hostname)` |
| `DisconnectCommand` | `[RelayCommand(CanExecute = nameof(CanDisconnect))]` | `ConnectionState == Connected` |

**Command re-evaluation**: `OnConnectionStateChanged` and `OnHostnameChanged` partial methods (generated by `[ObservableProperty]`) call `ConnectCommand.NotifyCanExecuteChanged()` and `DisconnectCommand.NotifyCanExecuteChanged()`.

### RdpClientHost (Interop вҖ” Interop/RdpClientHost.cs)

Adapted from `poc/StickyDesktop/RdpClientHost.cs`. Key changes from POC:

| Change | Reason |
|--------|--------|
| Implements `IRdpClient` | Enables ViewModel to consume via interface |
| Remove `Log()` method and file logging | Replace with no-op; structured logging is future scope |
| `ApplyDefaultSettings()` method | Encapsulates AdvancedSettings configuration (NLA, SmartSizing, SmartCards, AuthLevel, ColorDepth) |
| Production namespace `Remort.Interop` | Proper namespace per project structure |
| Remove `UserName`, `Domain` properties | Not needed вҖ” credential prompt handles auth (FR-012) |
| Remove `Ocx` public property | Leaks COM object; not needed by ViewModel |
