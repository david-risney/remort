# Contract: Auto-Reconnect Settings

**Feature**: 004-auto-reconnect-on-login
**Type**: Internal data contract (Settings persistence boundary)
**Files**: `src/Remort/Settings/AppSettings.cs`, `src/Remort/Connection/MainWindowViewModel.cs`

## Purpose

This contract defines the settings fields and ViewModel interface additions that support auto-reconnect on Windows login. The feature extends the existing `AppSettings` record (from spec 003) with two new fields and adds a public method and observable property to `MainWindowViewModel`.

No new interfaces or services are introduced. All additions build on existing infrastructure.

## AppSettings Record — New Fields

```csharp
// Added to existing AppSettings record in Settings/AppSettings.cs

/// <summary>Gets a value indicating whether auto-reconnect on Windows login is enabled.</summary>
public bool AutoReconnectEnabled { get; init; }

/// <summary>Gets the hostname of the last connected (or last attempted) host.</summary>
public string LastConnectedHost { get; init; } = string.Empty;
```

### Serialization

| Property | JSON Key | Type | Default | Notes |
|----------|----------|------|---------|-------|
| `AutoReconnectEnabled` | `autoReconnectEnabled` | `bool` | `false` | FR-006: disabled by default |
| `LastConnectedHost` | `lastConnectedHost` | `string` | `""` | FR-009: empty = no auto-reconnect |

**Backward compatibility**: Existing `settings.json` files missing these fields deserialize with defaults via `System.Text.Json`'s missing-property handling and `init` defaults on the record.

## MainWindowViewModel — New Members

### Observable Properties

```csharp
// Generated by CommunityToolkit.Mvvm [ObservableProperty]

/// <summary>Gets or sets a value indicating whether auto-reconnect on login is enabled.</summary>
[ObservableProperty]
private bool _autoReconnectEnabled;
```

Bound to a `CheckBox` in the connection bar. On change, persists to `ISettingsStore`:

```csharp
partial void OnAutoReconnectEnabledChanged(bool value)
{
    if (_settingsStore is not null)
    {
        AppSettings current = _settingsStore.Load();
        _settingsStore.Save(current with { AutoReconnectEnabled = value });
    }
}
```

### TryAutoReconnect Method

```csharp
/// <summary>
/// Attempts auto-reconnect if enabled, disconnected, and a last host is known.
/// Called by the View when a Windows session unlock/logon event occurs.
/// </summary>
public void TryAutoReconnect()
```

**Preconditions** (all must be true to proceed):
1. `ConnectionState == ConnectionState.Disconnected`
2. `AutoReconnectEnabled == true`
3. `LastConnectedHost` (from settings) is not null or empty

**Behavior**:
- Sets `_isAutoReconnect = true` (internal flag for status text formatting)
- Sets `Hostname` to the last connected host (for UI display)
- Sets `_connectionService.Server` to the last connected host
- Sets `ConnectionState = ConnectionState.Connecting`
- Calls `_connectionService.Connect()`

**If any precondition fails**: Returns silently (no error, no state change).

## IConnectionService / RetryingConnectionService — Unchanged

No modifications to the connection service interface or implementation. Auto-reconnect uses the identical `Connect()` / `Disconnect()` flow as manual connections. The retry policy (from spec 003) applies automatically.

## ISettingsStore / JsonSettingsStore — Unchanged

No modifications to the settings interface or implementation. The `AppSettings` record's new fields are handled by `System.Text.Json` serialization/deserialization automatically.

## Event Wiring — MainWindow.xaml.cs

```csharp
// In constructor, after ViewModel creation:
Microsoft.Win32.SystemEvents.SessionSwitch += OnSessionSwitch;

// Handler:
private void OnSessionSwitch(object sender, Microsoft.Win32.SessionSwitchEventArgs e)
{
    if (e.Reason == Microsoft.Win32.SessionSwitchReason.SessionUnlock
        || e.Reason == Microsoft.Win32.SessionSwitchReason.SessionLogon)
    {
        Dispatcher.Invoke(() => _viewModel.TryAutoReconnect());
    }
}

// In OnClosing:
Microsoft.Win32.SystemEvents.SessionSwitch -= OnSessionSwitch;
```

## Consumers

| Consumer | Usage |
|----------|-------|
| `MainWindow.xaml` | Binds `CheckBox.IsChecked` to `AutoReconnectEnabled` |
| `MainWindow.xaml.cs` | Calls `TryAutoReconnect()` on session unlock/logon |
| `MainWindowViewModel` | Reads/writes `AutoReconnectEnabled` and `LastConnectedHost` via `ISettingsStore` |

## Constraints

- `TryAutoReconnect()` must be safe to call multiple times (idempotent when already connecting/connected).
- `LastConnectedHost` is updated on connection *initiation*, not on success.
- The `CheckBox` for auto-reconnect is only enabled when disconnected (uses existing `DisconnectedToBoolConverter`).
- Auto-reconnect uses the same retry policy as manual connections — no separate retry configuration.
